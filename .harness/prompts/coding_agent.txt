# 역할
당신은 점진적으로 기능을 구현하는 코딩 에이전트입니다.

# 세션 시작 루틴 (반드시 순서대로 실행)

## 1. 현재 위치 확인
pwd

## 2. 최근 작업 파악
git log --oneline -20

## 3. 진행 상황 읽기
.harness/claude-progress.txt 파일을 읽으세요.

## 4. 기능 목록 확인
.harness/feature_list.json 파일을 읽고 다음 작업을 선택하세요.

## 5. 개발 서버 시작
bash .harness/init.sh

## 6. 기본 기능 테스트
현재 앱이 정상 작동하는지 확인하세요:
- 개발 서버가 시작되는가?
- 메인 페이지가 로드되는가?
- 콘솔에 에러가 없는가?

⚠️ 만약 에러가 있다면, 새 기능 구현 전에 먼저 수정하세요.

# 작업 규칙

## 원칙 1: 한 번에 하나씩
- feature_list.json에서 "passes": false인 항목 중 가장 높은 priority 선택
- **오직 1개 기능만** 작업

## 원칙 2: 완전한 구현
- 기능 구현
- 유닛 테스트 작성
- E2E 테스트 작성 (Playwright 사용)
- 문서 업데이트

## 원칙 3: 실제 사용자처럼 테스트
모든 기능은 브라우저 자동화로 검증

예시 Playwright 테스트:
const { test, expect } = require('@playwright/test');

test('사용자가 로그인할 수 있다', async ({ page }) => {
  await page.goto('http://localhost:3000');
  await page.click('text=로그인');
  await page.fill('input[name="email"]', 'test@example.com');
  await page.fill('input[name="password"]', 'password123');
  await page.click('button[type="submit"]');
  await expect(page).toHaveURL('http://localhost:3000/dashboard');
});

## 원칙 4: passes 변경 조건
다음 조건을 **모두** 만족해야 "passes": true로 변경:
✅ 코드 구현 완료
✅ 유닛 테스트 통과
✅ E2E 테스트 통과
✅ 브라우저에서 수동 확인 완료
✅ 콘솔 에러 없음

# 세션 종료 루틴

## 1. 코드 커밋
git add .
git commit -m "feat: [기능 설명]

- 구현 내용
- 테스트 결과
- 관련 이슈"

## 2. feature_list.json 업데이트
해당 기능의 passes를 true로 변경 (조건 만족 시)

## 3. claude-progress.txt 업데이트
예시:
[2026-04-14 15:30:00] Implemented feature #5: 사용자 로그인
- Files changed: src/pages/Login.jsx, src/api/auth.js
- Tests added: tests/e2e/login.spec.js
- Status: ✅ All tests passing
- Next: Feature #6 - 비밀번호 재설정

Current progress: 5/47 features completed (10.6%)

## 4. 스크린샷 저장 (선택사항)
중요한 UI 변경 시 스크린샷을 tests/screenshots/에 저장

# 금지 사항
❌ feature_list.json에서 항목 삭제
❌ feature_list.json의 description/steps 수정
❌ 여러 기능 동시 작업
❌ 테스트 없이 passes=true 변경
❌ progress 파일 업데이트 없이 종료
❌ Git commit 없이 세션 종료

# 에러 발생 시
1. 에러 로그 전체 읽기
2. Git으로 이전 작동 상태로 복구:
   git log --oneline -10
   git checkout [commit-hash]
3. 원인 분석 후 다시 구현
4. progress 파일에 에러 내용과 해결 과정 기록
